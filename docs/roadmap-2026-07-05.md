## 0. The through-line (read this first)

ClawCrate is not "a sandbox." The sandbox is commodity — every agent vendor (Codex, Cursor, Claude Code) already ships one internally, and kernel primitives (Landlock/seccomp/Seatbelt) are free. The defensible category is **Policy + Evidence for Agent Actions**: least-authority *policy* (profiles → `learn` → marketplace) + kernel *enforcement* + tamper-evident *evidence* (audit chain) — **agent-agnostic**, so it serves every agent instead of one.

Every milestone below advances one of those three layers or converts already-built capability into adoption. The sequence is deliberate: **capitalize what exists → make the security promise true everywhere → remove authoring friction → build the ecosystem → freeze the surface.**

## 3. Phase B — Foundations: Security Parity + Scalability (v0.3.0 · 2026-08 → 2026-09-30)

**Goal:** Make the core promise — *your agent cannot read your secrets* — true on **every platform and mode**, and pay down the structural debt that blocks all later features.

### 3.1 Security Parity & Hardening (Epic #268, p0)

The headline (`cat ~/.ssh/id_rsa → EACCES`) is today only enforced on macOS or Linux+Replica. **Linux Direct Mode does not mediate reads** — Landlock is configured with write-only access rights. This is the #1 product gap.

| Issue | Deliverable | Size | Prio |
| --- | --- | :-: | :-: |
| #272 | **Landlock read-allowlisting** — declare `ACCESS_FS_READ_*` in `handled_access_fs`, grant read only to `fs_read` + minimal system/toolchain set | L | p0 |
| #273 | seccomp deny-by-default allowlist (or documented + expanded denylist) | M | p1 |
| #274 | Artifact permissions `0700`/`0600` at creation | S | p1 |
| #275 | Guard non-loopback `api --bind` behind `--allow-remote-bind` | S | p2 |
| #276 | Surface the Linux Direct-Mode read gap in `plan`/`doctor` (zero-code mitigation, auto-removed when #272 lands) | S | p1 |

**Sequencing:** #276 first (immediate honest mitigation, zero code) → #274 (cheap, high-coherence with compliance) → #272 (the big one; the CI fixture asserting Linux read-isolation is the milestone gate) → #273 ‖ #275.

### 3.2 Scalability & Maintainability (Epic #269, p1)

| Issue | Deliverable | Size | Prio |
| --- | --- | :-: | :-: |
| #277 | Decompose `main.rs` (5.8k lines) into feature modules — no behavior change | M | p1 |
| #278 | Replica copy-on-write/reflink (`clonefile`/`FICLONE`) + hardlink fast path | M | p1 |
| #279 | Metadata-first `fs-diff` with on-demand hashing (keep audit-grade opt-in) | M | p2 |

**Sequencing:** #277 **first and early** — it is the soft blocker for `learn` and the Action, and lands cleanest before #272's changes create merge pressure. Then #278 (unlocks MCP wrap on large trees / `$HOME` filesystem servers) → #279.

**Exit criteria (v0.3.0):**
- Linux Direct Mode `safe`/`build` blocks `~/.ssh/id_rsa` reads; workspace + toolchain reads still work; CI fixture green on both platforms.
- seccomp posture is coherent with "deny by default" (allowlist) or explicitly documented best-effort + expanded.
- Artifacts are `0700`/`0600`; non-loopback API bind is gated.
- `main.rs` is thin wiring; golden/integration/security suites unchanged.
- Replica of a large tree is near-instant on CoW filesystems.

## 4. Phase C — Adoption Unlock: `clawcrate learn` (v0.4.0 · 2026-09 → 2026-11-30)

**Goal:** Remove the #1 friction of every sandbox — *writing the policy*. Trace a command's real behavior and synthesize the tightest-fit profile.

**Why it is the 0→1 lever:** it turns ClawCrate from "for devs who write YAML" into "for anyone with an agent," and it is the *enabler* of the Linux read-allowlist (#272) for custom profiles — `learn` discovers the exact read set a command needs. No competitor (sandbox-runtime, microsandbox, nono) has auto-policy from a trace.

**Workstreams (Epic #222):**

| Deliverable | Platform | Size |
| --- | --- | :-: |
| Trace harness Linux (`ptrace` / seccomp-unotify / eBPF): reads, writes, network, env, processes | Linux (Tier 1) | L |
| Profile synthesizer (trace → tightest-fit YAML) + minimal read-path cover algorithm | cross | M |
| `clawcrate learn` subcommand + "add to community catalog?" prompt | cross | S |
| Integration with #272: `learn` emits the Linux read-allowlist for custom profiles | Linux | S |
| macOS trace harness best-effort (DTrace/ESF) with honest SIP caveats | macOS (Tier 2) | M |

**Honest caveat:** macOS filesystem tracing needs SIP-off or entitlements a MIT binary cannot hold. Document Linux as first-class; recommend running `learn` on Linux and sharing the cross-platform profile.

**Exit criteria:** `clawcrate learn -- <cmd>` on Linux produces a runnable tightest-fit profile that then runs green under `clawcrate run`.

## 5. Phase D — Ecosystem & Distribution (v0.4.0 · parallel/after C)

**Goal:** Turn policy from a per-user cost into a shared asset (network effect), and move the ICP from "individual" to "team with audit in every PR."

### 5.1 Marketplace (Epic #221)
`clawcrate profiles fetch/search/install` against an MVP catalog (GitHub raw + SHA256), bootstrap the external `clawcrate/profiles` repo, seed the top-50 tool profiles, and define sigstore signing for a later phase. **Consumes `learn`'s output** — learned profiles are the seed supply, which is why marketplace follows learn.

### 5.2 Distribution (Epic #220)
`clawcrate/action@v1` (upload-audit, fail-on-tampering via `verify`), reference integration recipes (Claude Code, Codex, Cursor, Aider, **OpenClaw**, Continue.dev), and a VS Code extension MVP (run-under-clawcrate + audit panel + status bar). Homebrew tap + `cargo install`.

> **Highest-leverage distribution bet (from the audit §3.2):** a **first-class OpenClaw integration** — a hook/plugin that routes OpenClaw's shell execution and MCP-server launches through ClawCrate by default. OpenClaw is our largest installed-base surface; inheriting it beats building an audience. Prioritize #220's OpenClaw recipe into a real integration, not just a doc.

**Exit criteria:** `clawcrate profiles install <name>` works against the MVP catalog with SHA256 verification; the GitHub Action uploads and verifies audit artifacts; at least one first-class agent integration (OpenClaw) exists beyond a doc recipe.

## 6. Phase E — Structural Hardening & Surface Freeze (v1.0 · 2027 Q1)

**Goal (Epic #271):** production-grade hardening and a frozen surface.

- **Structural network enforcement** — move `network: filtered` from proxy/env best-effort to enforced (loopback-only egress + backend block of direct connections, or DNS interception). Closes the residual bypass in `docs/egress-proxy-threat-model.md`.
- **`clawcrate replay <run-id>`** — reproduce a recorded execution from plan + artifacts.
- **Windows / WSL2 go/no-go** — decide native support on ≥3 explicit real-user requests; WSL2 covers the baseline.
- **Compliance certifications** — SOC2 Type 1 readiness + published EU AI Act statement.
- **Plugin system** — design surface in v0.3.0, deliver in v1.0.
- **`sandbox-exec` contingency** — ESF backend research + documented alcless-style user-isolation fallback.

## 8. Consolidated differentials (the narrative to defend)

**Defensible today (shipped in `main` / `v0.2.0-alpha.0`):**
1. **Only standalone dual-platform native sandbox** — Landlock+seccomp / Seatbelt, no Docker, no root, single Rust binary. (Others are agent-internal, Docker-based, or macOS-only.)
2. **Audit-grade by default, MIT** — SHA-256 hash chain + canonical JSON (RFC 8785) + offline `verify` + Ed25519 signing + SIEM export. Matches closed-source `nono`; beats agents' plain logs.
3. **MCP Server Firewall** — transparently sandbox any stdio MCP server; the client never notices. Nobody else does this.
4. **Agent-agnostic** — the same binary serves OpenClaw, Claude Code, Codex, Cursor, Gemini CLI, and CI. A neutrality no agent vendor can copy without ceasing to be an agent vendor.
5. **Concrete compliance narrative** — EU AI Act Art. 12/19/26 mapping + IETF draft-sharif alignment. Turns "security" into "regulatory evidence," a purchase driver.

**Under construction (the moat):** least-authority policy that authors itself (`learn`, Phase C) and becomes a shared, signed, reusable asset (`marketplace`, Phase D). Umbrella positioning: **"the governance layer for agent-executed commands — minimal authority + portable evidence, for any agent."**
