## GitHub Sync Policy

- Open small GitHub issues only from entries already present in
  `docs/backlog.yaml`.
- When opening a small issue, copy the backlog title, labels, milestone, and
  scope.
- After opening the issue, update `github_issue` from `null` to the new issue
  number.
- When a PR merges, ensure the corresponding issue is closed and update the
  backlog state from `planned` to `closed`.
- Close an Epic only when all child backlog items for that Epic are closed or
  explicitly removed from scope.
